# Screenshots Guide

This directory should contain screenshots of all app screens referenced in the main README.

## Required Screenshots

Please run the app and take screenshots of the following screens:

### Onboarding & Authentication (3 screenshots)
- `onboarding.png` - The onboarding/welcome screen
- `login.png` - Login screen
- `signup.png` - Signup screen

### Discover Merchants (2 screenshots)
- `map-home.png` - Main map view showing nearby merchants
- `merchant-detail.png` - Individual merchant details view

### Merchant Dashboard (3 screenshots)
- `register-merchant.png` - Merchant registration form
- `merchant-dashboard.png` - Merchant dashboard home
- `business-details.png` - Business details management screen

### Payments (3 screenshots)
- `scan-qr.png` - QR code scanner screen
- `request-payment.png` - Payment request screen
- `payment-success.png` - Payment success confirmation

### Profile (2 screenshots)
- `profile.png` - User profile screen
- `saved-restaurants.png` - Saved/favorite merchants list

## How to Take Screenshots

1. **Run the app**:
   ```bash
   cd Mobileapp
   npx expo start
   ```

2. **Navigate to each screen** listed above

3. **Take screenshot**:
   - iOS Simulator: Cmd+S or File → Save Screen
   - Android Emulator: Click camera icon in toolbar
   - Physical device: Use device screenshot function

4. **Save with exact filename** listed above

5. **Place in this directory**: `screenshots/`

## Screenshot Requirements

- Format: PNG
- Orientation: Portrait
- Size: Native device resolution (will be resized in README)
- Content: Clean UI without debug overlays

After adding screenshots, you can delete this guide file.
